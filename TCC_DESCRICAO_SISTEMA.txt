RECORTE DE APRESENTACAO
Este documento descreve apenas o modulo de impressoras, com foco em coleta automatizada de dados e exibicao operacional no sistema web.

1. VISAO GERAL DO MODULO DE IMPRESSORAS

O modulo de impressoras foi desenvolvido para substituir verificacoes manuais por um fluxo automatizado de coleta, armazenamento e visualizacao de dados operacionais.

Objetivo do recorte:
- Coletar dados tecnicos das impressoras periodicamente.
- Exibir status e indicadores em uma interface unica para a equipe de TI.
- Apoiar a gestao de suprimentos com informacao atualizada e historica.


2. COMO FUNCIONA O FLUXO (COLETA E EXIBICAO)

2.1 Coleta automatizada
- Um coletor em Python consulta equipamentos de impressao via SNMP.
- As leituras capturam dados de operacao e consumo (telemetria/pagecount, status e observacoes).
- O coletor executa em ciclos, registrando historico para analise e acompanhamento.

2.2 Persistencia dos dados
- Os dados coletados sao armazenados em banco relacional (Supabase/PostgreSQL).
- O historico permite comparacao temporal, identificacao de tendencia e suporte a decisao.

2.3 Exibicao web
- O frontend em Next.js/React apresenta visoes operacionais de impressoras.
- A equipe de TI visualiza rapidamente status, itens criticos e atualizacoes.
- O foco da tela e acao operacional: identificar problema e agir.


3. ARQUITETURA DO MODULO

3.1 Camadas
- Coleta: Python + SNMP.
- Dados: Supabase (PostgreSQL).
- APIs operacionais: Supabase Edge Functions.
- Interface: Next.js (hospedado no Vercel).

3.2 Papel de cada camada
- Coletor: captura informacao dos dispositivos de impressao.
- Banco: centraliza, organiza e historiza os registros.
- Edge Functions: aplicam regras e disponibilizam dados para o frontend.
- Frontend: transforma dados tecnicos em visao operacional para uso diario.

5. ESTRUTURA DE DADOS RELACIONADA AO MODULO DE IMPRESSORAS

Principais entidades utilizadas neste recorte:

5.1 Inventario (base dos equipamentos)
- Tabela: inventario
- Funcao: representa a instancia fisica do equipamento (incluindo impressoras no parque).
- Campos relevantes no recorte: nr_inventario, nr_patrimonio, nr_serie, nr_ip, tp_status.

5.2 Telemetria de impressao
- Tabela: telemetria_pagecount
- Funcao: historico de contadores e estado operacional.
- Campos relevantes: nr_paginas_total, nr_paginas_coloridas, nr_paginas_pb, dt_leitura, ds_status_impressora.

5.3 Suprimentos
- Tabela: suprimentos
- Funcao: controle de niveis e limites de insumos por equipamento.
- Campos relevantes: tp_suprimento, nr_quantidade, nr_quantidade_minima, nr_quantidade_maxima.

5.4 Setor
- Tabela: setor
- Funcao: localizacao organizacional do equipamento para apoio operacional.

5.5 Movimentacao (apoio)
- Tabela: movimentacao
- Funcao: historico de mudanca de local/setor do equipamento.


6. REGRAS OPERACIONAIS NO CONTEXTO DE IMPRESSORAS

- Coleta periodica para reduzir dependencia de verificacao manual.
- Registro historico para rastreabilidade e auditoria tecnica.
- Exibicao consolidada para priorizacao de atendimento.
- Status operacional padronizado (ATIVO, MANUTENCAO, BACKUP, DEVOLUCAO), quando aplicavel ao item monitorado.


7. RESULTADO PRATICO ESPERADO COM ESTE MODULO

- Reducao do tempo gasto em rondas e consultas manuais de impressoras.
- Aumento da produtividade da equipe de TI.
- Maior previsibilidade de reposicao de suprimentos.
- Menor risco de indisponibilidade por falta de insumo.
- Melhoria da governanca operacional por dados historicos.


8. DELIMITACAO DO ESCOPO DE APRESENTACAO

Este recorte considera exclusivamente:
- Coleta de dados de impressoras (SNMP).
- Armazenamento e historico operacional.
- Exibicao das informacoes para suporte a decisao da equipe de TI.

Nao faz parte desta apresentacao:
- Modulos de conciliacao Matrix x Inventario.
- Cadastros amplos de ativos nao relacionados ao foco de impressoras.


9. CONCLUSAO TECNICA DO RECORTE

O modulo de impressoras demonstra, de forma objetiva, como a automacao pode melhorar a eficiencia operacional em ambiente hospitalar. A combinacao de coleta automatizada, base historica e exibicao centralizada reduz trabalho manual, aumenta visibilidade operacional e melhora a capacidade de resposta da equipe de TI.
